## Cross-Package Generic Conversions

**NEW**: Convert between generic types across different packages with full type resolution.

### Basic Cross-Package Setup

**Project Structure:**
```
project/
├── internal/
│   └── models/
│       └── user.go      # domain.User
├── api/
│   └── dto/
│       └── user.go      # dto.UserDTO
└── converters/
    └── user_converter.go # Generic converter
```

**Define Generic Converter:**

```go
//go:build convergen

package converters

// Generic converter interface
type UserConverter[S, D any] interface {
    // :typecast
    // :struct-literal
    Convert(S) D
    ConvertSlice([]S) []D
}
```

**Generate with CLI:**

```bash
# Cross-package generation with type resolution
convergen -type 'UserConverter[models.User,dto.UserDTO]' \
          -imports 'models=./internal/models,dto=./api/dto' \
          -struct-literal \
          user_converter.go
```

**Generated Code:**

```go
// Code generated by github.com/reedom/convergen
// DO NOT EDIT.

package converters

import (
    "github.com/yourorg/project/internal/models"
    "github.com/yourorg/project/api/dto"
)

func Convert(src models.User) dto.UserDTO {
    return dto.UserDTO{
        ID:   int64(src.ID),
        Name: src.Name,
        Email: src.Email,
        CreatedAt: src.CreatedAt,
    }
}

func ConvertSlice(src []models.User) []dto.UserDTO {
    dst := make([]dto.UserDTO, len(src))
    for i, item := range src {
        dst[i] = Convert(item)
    }
    return dst
}
```

## Supported Generic Constraints

Convergen supports all major Go generic constraint types:

### Any Constraint

```go
type Converter[T, U any] interface {
    Convert(T) U
    ConvertPointer(*T) *U
    ConvertSlice([]T) []U
}
```

### Comparable Constraint

```go
type IDConverter[T comparable, U comparable] interface {
    // :map String() Value
    ConvertID(ID[T]) ID[U]
}

type ID[T comparable] struct {
    Value T
}

func (id ID[T]) String() string {
    return fmt.Sprintf("%v", id.Value)
}
```

### Union Constraints

```go
type NumberConverter[T ~int | ~int32 | ~int64, U ~float32 | ~float64] interface {
    // Automatic type casting between numeric types
    // :typecast
    ConvertNumber(T) U
    ConvertSlice([]T) []U
}
```

### Interface Constraints

```go
type StringerConverter[T fmt.Stringer, U interface{ SetString(string) }] interface {
    // :map String() StringValue
    ConvertStringer(T) U
}

// Usage example
type Status int
func (s Status) String() string { return "status" }

type StatusDTO struct {
    StringValue string
}
func (s *StatusDTO) SetString(value string) { s.StringValue = value }
```

## Real-World Generic Patterns

### Repository Pattern with Generics

```go
// Domain models
type Repository[T any] struct {
    items []T
    metadata map[string]interface{}
}

// API DTOs
type RepositoryDTO[U any] struct {
    Items []U `json:"items"`
    Count int  `json:"count"`
}

// Generic converter
type RepoConverter[T, U any] interface {
    // :map items Items
    // :literal Count len(src.items)
    ConvertRepo(*Repository[T]) *RepositoryDTO[U]
}
```

**CLI Usage:**

```bash
convergen -type 'RepoConverter[models.Product,dto.ProductDTO]' \
          -imports 'models=./internal/models,dto=./api/dto' \
          repo_converter.go
```

### Collection Transformations

```go
type CollectionConverter[T, U any] interface {
    // Basic conversions
    ConvertSlice([]T) []U
    ConvertMap(map[string]T) map[string]U

    // With filtering
    // :conv filterValid src
    ConvertFiltered([]T) []U

    // Pagination support
    // :literal Page src.Page
    // :literal Total len(src.Items)
    ConvertPaginated(*PaginatedList[T]) *PaginatedList[U]
}

type PaginatedList[T any] struct {
    Items []T
    Page  int
    Size  int
}

func filterValid[T Validatable](items []T) []T {
    var result []T
    for _, item := range items {
        if item.IsValid() {
            result = append(result, item)
        }
    }
    return result
}
```

### Type-Safe ID Conversions

```go
// Generic ID types
type UserID int64
type OrderID string
type ProductID uuid.UUID

// Generic ID wrapper
type TypedID[T comparable] struct {
    Value T `json:"id"`
    Type  string `json:"type"`
}

// Converter for different ID types
type IDConverter[T comparable, U comparable] interface {
    // :map Value.String() StringValue
    // :literal Type "converted"
    ConvertTypedID(*TypedID[T]) *TypedID[string]

    // Direct ID conversion
    // :conv convertID Value
    ConvertDirectID(T) U
}

func convertID[T, U comparable](src T) U {
    // Implementation depends on types
    return *new(U)
}
```

## Advanced Generic Features

### Nested Generic Types

```go
type Result[T any] struct {
    Data  T
    Error error
}

type Response[U any] struct {
    Payload U      `json:"payload"`
    Success bool   `json:"success"`
    Message string `json:"message,omitempty"`
}

type ResultConverter[T, U any] interface {
    // :map Data Payload
    // :literal Success src.Error == nil
    // :map Error.Error() Message
    ConvertResult(*Result[T]) *Response[U]
}
```

### Generic Builder Pattern

```go
type Builder[T any] struct {
    config map[string]interface{}
    result T
}

type BuilderDTO[U any] struct {
    Configuration map[string]interface{} `json:"config"`
    Output       U                      `json:"output"`
}

type BuilderConverter[T, U any] interface {
    // :map config Configuration
    // :map result Output
    ConvertBuilder(*Builder[T]) *BuilderDTO[U]
}
```

## Error Handling and Validation

### Constraint Validation

Convergen validates generic constraints at generation time:

```bash
# Invalid constraint usage
$ convergen -type 'Converter[invalid.Type,dto.Valid]' \
           -imports 'invalid=./nonexistent,dto=./dto' \
           converter.go

Error: cannot resolve package "invalid" at path "./nonexistent"
Suggestion: Check the package path and ensure it exists

# Type compatibility validation
$ convergen -type 'Converter[models.User,dto.Product]' \
           -imports 'models=./models,dto=./dto' \
           converter.go

Warning: Type compatibility check shows potential field mismatches
Suggestion: Review field mappings between models.User and dto.Product
```

## Best Practices

### Generic Interface Design

```go
// ✅ Good: Clear type parameters with meaningful constraints
type UserConverter[S User, D UserDTO] interface {
    Convert(S) D
}

// ❌ Avoid: Overly broad constraints without meaning
type Converter[S, D any] interface {
    DoEverything(S) D
}
```

### Package Organization

```go
// ✅ Good: Organize by domain boundaries
converters/
├── user_converter.go     # User domain conversions
├── order_converter.go    # Order domain conversions
└── common_converter.go   # Shared utilities

// ❌ Avoid: Single massive converter file
converters/
└── all_converters.go     # Everything mixed together
```

### Error Handling

```go
// ✅ Good: Explicit error handling for conversions that can fail
type Converter[S, D any] interface {
    Convert(S) (D, error)
    ConvertUnsafe(S) D  // For conversions that never fail
}

// ❌ Avoid: Ignoring potential conversion errors
type Converter[S, D any] interface {
    Convert(S) D  // What if conversion fails?
}
```

## Troubleshooting

### Common Issues

**Type Resolution Failures:**
- Verify package paths are correct
- Check import aliases match type specifications
- Ensure target packages compile independently

**Constraint Violations:**
- Review generic constraint compatibility
- Validate type parameter usage
- Check for circular dependencies

**Performance Issues:**
- Use caching for repeated type resolution
- Consider parallel processing for large codebases
- Monitor memory usage during generation
