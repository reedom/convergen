    Quick Start¶

    Get up and running with Convergen in 5 minutes! This guide walks you through creating your first conversion function from scratch.

    Overview¶

    In this tutorial, you'll:

    1. Set up a simple Go project
    2. Define source and destination types
    3. Create a conversion interface with annotations
    4. Generate the conversion function
    5. Use the generated code in your application

    Step 1: Project Setup¶

    Create a new Go project:

    mkdir convergen-tutorial
cd convergen-tutorial
go mod init github.com/yourusername/convergen-tutorial

    Step 2: Define Your Types¶

    Let's create a common scenario: converting between domain models and API responses.

    Create types.go:

    package main

import "time"

// Domain model - your internal representation
type User struct {
    ID        int
    FirstName string
    LastName  string
    Email     string
    Password  string // Sensitive field we don't want in API
    CreatedAt time.Time
    Status    UserStatus
}

type UserStatus int

const (
    StatusInactive UserStatus = iota
    StatusActive
    StatusSuspended
)

func (s UserStatus) String() string {
    switch s {
        case StatusActive:
            return "active"
        case StatusSuspended:
            return "suspended"
        default:
            return "inactive"
    }
}

// API response - what clients receive
type UserResponse struct {
    ID          int    `json:"id"`
    FirstName   string `json:"first_name"`
    LastName    string `json:"last_name"`
    Email       string `json:"email"`
    Status      string `json:"status"`
    CreatedDate string `json:"created_date"`
}

    Step 3: Create Converter Interface¶

    Create converter.go:

    //go:build convergen

package main

//go:generate go run github.com/reedom/convergen@v8.0.3
type Convergen interface {
    // Convert User to UserResponse
    // :skip Password                           # Don't copy sensitive field
    // :stringer                                # Use String() method for Status
    // :conv formatTimestamp CreatedAt CreatedDate # Use converter for timestamp formatting
    UserToResponse(*User) *UserResponse
}

    Step 4: Generate Code¶

    Run the code generator:

    go generate ./...

    This creates converter.gen.go with your conversion function:

    // Code generated by github.com/reedom/convergen
// DO NOT EDIT.

package main

func UserToResponse(src *User) (dst *UserResponse) {
    dst = &UserResponse{}
    dst.ID = src.ID
    dst.FirstName = src.FirstName
    dst.LastName = src.LastName
    dst.Email = src.Email
    dst.Status = src.Status.String()
    dst.CreatedDate = formatTimestamp(src.CreatedAt)
    return
}

// Custom converter function (you need to define this)
func formatTimestamp(t time.Time) string {
    return t.Format("2006-01-02T15:04:05Z07:00")
}

    Step 5: Use the Generated Code¶

    Create main.go:

    package main

import (
    "encoding/json"
    "fmt"
    "time"
)

func main() {
    // Create a sample user
    user := &User{
        ID:        1,
        FirstName: "John",
        LastName:  "Doe",
        Email:     "john.doe@example.com",
        Password:  "secret123", // This won't appear in the API response
        CreatedAt: time.Now(),
        Status:    StatusActive,
    }

    // Convert to API response using generated function
    response := UserToResponse(user)

    // Serialize to JSON
    jsonData, err := json.MarshalIndent(response, "", "  ")
    if err != nil {
        panic(err)
    }

    fmt.Println("User converted to API response:")
    fmt.Println(string(jsonData))
}

    Step 6: Run and Test¶

    go run .

    Output: 

    User converted to API response:
{
  "id": 1,
  "first_name": "John",
  "last_name": "Doe",
  "email": "john.doe@example.com",
  "status": "active",
  "created_date": "2024-01-15T10:30:00Z07:00"
}

    Notice how: - ✅ Password field was skipped (security) - ✅ Names mapped directly (FirstName, LastName) - ✅ Status enum was converted to string - ✅ Timestamp was formatted for JSON - ✅ No boilerplate code was written manually

    What Just Happened?¶

    1. Annotations Explained¶

    Annotation Purpose Result
    :skip Password Exclude sensitive field Password not copied
    :stringer Use String() method Status becomes "active"
    :conv formatTimestamp CreatedAt CreatedDate Custom converter for timestamp ISO 8601 timestamp

    2. Generated Function Features¶

    • Type-safe: Compile-time validation of conversions
    • Efficient: Direct field assignments, no reflection
    • Maintainable: Update annotations, regenerate code
    • Readable: Clean, understandable generated code

    3. Build Integration¶

    The //go:build convergen tag ensures: - Converter definitions don't interfere with regular builds - Clean separation between generation and runtime code - Standard Go tooling integration

    Next Steps: Common Patterns¶

    Error Handling¶

    For conversions that can fail:

    type Convergen interface {
    // :conv validateEmail Email
    UserToResponse(*User) (*UserResponse, error)
}

func validateEmail(email string) (string, error) {
    if !strings.Contains(email, "@") {
        return "", fmt.Errorf("invalid email: %s", email)
    }
    return email, nil
}

    Reverse Conversion¶

    Convert back from API to domain:

    type Convergen interface {
    // API to domain conversion
    // :literal Password ""                    # Set empty password
    // :literal CreatedAt time.Now()           # Set current time
    // :conv parseStatus Status                # Parse status string
    ResponseToUser(*UserResponse) *User
}

func parseStatus(status string) UserStatus {
    switch status {
    case "active":
        return StatusActive
    case "suspended":
        return StatusSuspended
    default:
        return StatusInactive
    }
}

    Multiple Interfaces¶

    Organize by purpose:

    // :convergen
type APIConvergen interface {
    UserToResponse(*User) *UserResponse
    ResponseToUser(*UserResponse) *User
}

// :convergen  
type StorageConvergen interface {
    UserToEntity(*User) *storage.UserEntity
    EntityToUser(*storage.UserEntity) *User
}

    Troubleshooting Quick Start¶

    Generation Fails?¶

    # Check Go version (1.21+ required)
go version

# Verify syntax
go build -o /dev/null ./...

# Clean and regenerate
rm -f *.gen.go
go generate ./...

    Code Won't Compile?¶

    • Check imports: Ensure all referenced types are accessible
    • Verify annotations: Review syntax against annotations reference
    • Validate functions: Ensure custom converter functions exist

    Unexpected Results?¶

    • Enable logging: Use convergen -log -print converter.go to debug
    • Check generated code: Review *.gen.go files to understand output
    • Test incrementally: Start simple, add complexity gradually
