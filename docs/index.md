# Convergen

**High-performance Go code generator for type-to-type copy functions**

[![Go Reference](https://pkg.go.dev/badge/github.com/reedom/convergen.svg)](https://pkg.go.dev/github.com/reedom/convergen) [![Go Report Card](https://goreportcard.com/badge/github.com/reedom/convergen)](https://goreportcard.com/report/github.com/reedom/convergen) ![Coverage](https://img.shields.io/badge/Coverage-67.4%25-yellow)

Convergen generates functions that copy field to field between two types, eliminating boilerplate code and reducing errors in type conversions.

## Quick Example

Write an annotated interface:

```go
//go:build convergen

package sample

import (
    "time"
    "github.com/sample/myapp/domain"
    "github.com/sample/myapp/storage"
)

//go:generate go run github.com/reedom/convergen@v9.0.0-beta.1
type Convergen interface {
    // :typecast
    // :stringer
    // :map Created.UnixMilli() Created
    DomainToStorage(*domain.User) *storage.User
}
```

Convergen generates optimized conversion functions:

```go
// Code generated by github.com/reedom/convergen
// DO NOT EDIT.

package sample

import (
    "time"
    "github.com/sample/myapp/domain"
    "github.com/sample/myapp/storage"
)

func DomainToStorage(src *domain.User) *storage.User {
    return &storage.User{
        ID:      int64(src.ID),
        Name:    src.Name,
        Status:  src.Status.String(),
        Created: src.Created.UnixMilli(),
    }
}
```

## Key Features

### 🎯 **Powerful Annotations**
Comprehensive annotation system for fine-grained control over conversions:

- **Field matching**: `:match name|none` - Control how fields are matched
- **Type conversions**: `:typecast`, `:stringer` - Handle type differences elegantly
- **Custom mappings**: `:map src dst` - Map fields with different names
- **Custom converters**: `:conv func src dst` - Use custom conversion functions
- **Flexible styles**: `:style return|arg` - Control function signatures

## Getting Started

Choose your preferred installation method:

=== "go:generate (Recommended)"

    Add to your Go file:
    ```go
    //go:generate go run github.com/reedom/convergen@v9.0.0-beta.1
    ```
    
    Then run:
    ```bash
    go generate ./...
    ```

=== "CLI Tool"

    Install globally:
    ```bash
    go install github.com/reedom/convergen@latest
    ```
    
    Use directly:
    ```bash
    convergen your-file.go
    ```
    
    By default, generates: `your-file.gen.go`

=== "Go Module"

    Add to your project:
    ```bash
    go get -u github.com/reedom/convergen@latest
    ```

## Use Cases

### Domain-to-Storage Conversion
Convert between domain models and storage representations:

```go
type Convergen interface {
    // :typecast
    UserToStorage(*domain.User) *storage.User
    StorageToUser(*storage.User) *domain.User
}
```

### API Response Mapping
Transform internal models to API responses:

```go
type Convergen interface {
    // :skip Password
    // :map CreatedAt.Format("2006-01-02") CreatedDate
    UserToResponse(*model.User) *api.UserResponse
}
```

### Microservice Communication
Convert between service boundaries:

```go
type Convergen interface {
    // :conv proto.Marshal Data
    ModelToProto(*internal.Model) (*pb.Model, error)
}
```

## Why Convergen?

| Problem | Without Convergen | With Convergen |
|---------|-------------------|----------------|
| **Boilerplate Code** | Hundreds of manual field assignments | Generated automatically |
| **Type Safety** | Runtime errors from missed fields | Compile-time validation |
| **Maintenance** | Update multiple conversion functions | Single annotation change |
| **Performance** | Generic reflection-based solutions | Optimized static code |
| **Testing** | Test every conversion function | Test annotations once |
